═══════════════════════════════════════════════════
PROBLEM STATEMENT
═══════════════════════════════════════════════════
The 32-cell DCM models a facility as a SINGLE tank: one age, one wall, full reset
on replace. T012/T013 showed this abstraction is violated by multi-tank portfolio
dynamics in ways BOY-stamping (Ticket 013) cannot fully fix:
  - F_replace assumes reset to age bin 1, but a one-tank swap at a multi-tank
    facility does not reset the facility's average age (T013 D1: 30% of replace
    transitions land off-support).
  - A_bin is a portfolio AVERAGE, so it moves non-monotonically as tanks are added
    or pruned, even under BOY stamping (cross-year composition changes).
  - The partial-closure margin (close some tanks, keep operating) is folded into
    Maintain and cannot be priced (T011/T012: the FF->RB response is size-dependent,
    +6.2pp at 4+ tanks, sign-reversing vs 2-3 tanks).

A faithful model represents the facility's PORTFOLIO: the joint distribution of
(tank count, per-tank age, per-tank wall), with actions that operate on individual
tanks (retrofit one, close one, install one) rather than a single facility-level
maintain/exit/replace.

═══════════════════════════════════════════════════
DESIGN QUESTIONS TO RESOLVE (architect, before any implementation)
═══════════════════════════════════════════════════
1. STATE: full portfolio (intractable) vs a sufficient summary. Candidate summary
   state: (n_tanks bin, count SW, count DW, age-profile summary). What is the
   minimal sufficient statistic for the CF of interest?
2. ACTIONS: per-tank {maintain, close, retrofit} + facility install, vs a facility
   action vector. How does the partial-closure 4th action (Option A) sit inside this?
3. TRANSITIONS: with per-tank actions, size/composition transitions become
   ENDOGENOUS (generated by the model's own choice probabilities) — this is the
   internally-consistent version that Ticket 015's exogenous fixed-type size avoids.
4. WELFARE: per-tank externality and replacement subsidy aggregation; the partial-
   closure social cost the current model cannot express.
5. IDENTIFICATION: does the per-tank panel (panel_dt) carry enough variation to
   identify per-tank retrofit/close costs separately? (Tie to the identification
   appendix.)

FORK RESOLVED (researcher, 2026-06-01) — see memory project_size_state_design:
  - Size is a FIXED-TYPE state dimension (32 -> 128 cells): block-diagonal F
    (no kernel; T012 stickiness). Assigned from BOY stock.
  - It enters the payoffs via SIZE-VARYING PRIMITIVES: re-aggregate P_vec, h_vec,
    hazard_loss, L_vec by size bin (researcher confirmed P and h move with size).
    Primitives PERSIST in the CF, so this is the CF-relevant channel through the
    common gamma.
  - PLUS a size FE (additive stay-margin intercept, profiled like the geographic
    stay-FE, Semantic 2 -> dropped in CF) for residual size heterogeneity / clean theta.
  - NOT size-specific kappa/K (deferred).
CAVEAT for the 015 design + a prerequisite diagnostic: P/h enter u_M only and the
size FE is dropped in the CF, so size variation on the EXIT/REPLACE margins
(kappa/K-governed) would be lost from the CF. Before specing 015, run the
"how does size enter" diagnostic on the BOY-corrected panel: confirm size-varying
P/h + common kappa/K reproduces the size-conditional CCPs on exit/replace; if not,
add size-specific kappa_size/K_size.
